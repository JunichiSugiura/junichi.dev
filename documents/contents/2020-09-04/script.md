## Intro
- 自分はLedgerという会社で暗号通貨用のハードウェア端末を提供している会社で働いている
- 自分が業務で書くコードはほぼ100%オープンソース
- 昨年末に就活をする際にもOSSの会社を真っ先に重視
- 趣味のコードもほぼオープンソースか最終的にpublicに公開
- 殆どのエンジニアは開発業務をクローズドソースで行っていると思う
- 何で"タダ"でコードを書くのか?
- OSSにすることでの現実的なコストを紹介

###  メリット

#### 使用する側のメリット
- 車輪の再発明をしない
  - 自分と同じ課題をもってる人は世界に絶対存在する
  - 時間の節約
  - 素早くideaをブートストラップ
- 自分のニーズに合わせて自由に改良できる
- 新しい出会い
  - 新しいプロダクト、アイディア  
  - 初対面の人に自分のことを説明しやすくなる
    - devコミュニティーでの自分のレピュテーションを気づける
      - 就活
      - 新しくプロジェクトを始めるとき
  - コラボをする中で新しい出会いがある
- キャリアに活かせる
  - 働き方のパラダイムシフト
    - 目的毎にで才能をもった個人が集まってプロジェクトを進めていく
    - 所属している企業より個人を重視
  - レピュテーション
  - スキルの証明
    - 技術スキル
    - コミュニケーションスキル
  - 自分の目的に近い会社を見つけられる

#### OSSとして公開するメリット
- コミュニティー
  - 世界中の才能
  - 多くの目に触れられる
  - よりよいidea
  - 汎用性の向上
    - Locked-inからくる技術負債の回避
  - セキュリティーの向上
    - バグの早期発見、対策
- needsの検証
  - 実装には時間がかかる
  - idea段階でのfeedbackを通してマーケットフィットの可能性を事前に検証できる
- competitor -> collaborator
- ブランディングに繋がる
  - ファンの獲得
    - コミュニティーの価値
    - ロイヤリティ
  - 消費者からの信用
    - セキュリティー
    - 透明性
      - 会社が公言しているようにシステムが動いていることの証明
  - 採用
    - 多くのdevはHR関連のメッセージにうんざりしている
      - 需要 > 供給
    - カルチャーフィット
      - お互いの目的
- 事例
  - Linux
    - 目的に応じた多彩なdistro
      - サーバー
        - セキュア
        - 軽量
      - desktop
        - GUI
    - RedHat
  - Microsoft
    - 過去の過ち
    - GitHub
    - VSCode
  - Facebook
    - 競合への圧力や個人データの収集など懸念もたくさんある会社だが憎めない点も多い
    - React
      - Web開発
      - Web以外への応用
        - MSの関与
  - HashiCorp
  - 挙げたらきりがない
  - Ledger
    - セキュリティー
    - 透明性
    - コラボレーション
      - コイン、チェーン
      - プラットフォーム
        - Brave
    - コミュニケーションは課題

#### コスト
- セキュリティ
  - 秘密鍵などの管理に一工夫必要
  - 重大なバグのコミュニケーション
- コミュニケーション
  - 現状をクリアに伝える
    - ツール
      - docs
      - issue
      - project board
      - chatroom (slack, discord, etc.)
      - conference talk
      - video
    - 何を目的としたプロジェクトなのか
      - 現状何ができるのか
    - 今どこの開発にフォーカスしているのか
    - 認知しているバグ
    - 重複したissueの整理
  - コミュニティーマネージャー
- ライセンス
  - sustainability
    - funding
